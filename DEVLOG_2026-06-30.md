# Part I — the logging gap  ·  8e7a0ad

`server.py` and `interpretation.py` — the two modules that actually run in
production — had **zero logging**. Every other module follows a
`logging.getLogger("fb.<name>")` + `StreamHandler(stdout)` + INFO convention
(`deal_truth.py`, `tools.py`, `trust_engine.py`, …); these two just never adopted
it. Meanwhile `deal_truth._build_guardrails` already detects exactly what the
review asked for — missing non-negotiables, source conflicts, unreconciled exit
value, non-standard multiples — and `server.py` already mints a UUID per upload
(`sid`) and per chat session (`session_id`). None of it was ever logged together.

Fix: added `fb.server` / `fb.interpretation` loggers (same convention, no new
pattern). Threaded `analysis_id` (the existing `sid`) through
`assemble_fact_sheet()` and `build_investment_read()` as an optional kwarg — no
contextvars, no signature changes deeper than those two entry points. Now every
guardrail message, every data-confidence tier, and every narration-source
decision (`gpt` / `deterministic` / `deterministic-after-exception`) logs tagged
with the analysis it belongs to. Verified live against Westview: 13 guardrails
logged on one run, e.g. `"Sources disagree on debt; use ONLY the canonical
winner 39000000.00000002 from Monthly Cash Flow!I138"` — previously computed,
passed to GPT, and never seen again.

# Part II — a confidence signal that never left the building  ·  8e7a0ad

`assemble_fact_sheet` computes `fs["confidence"]["data_coverage"]` — whether
NOI/opex/revenue are T1 (validated cash-flow), T2-footed (components sum to the
total), or T2-unfooted (a single line, not a footed total) — but
`render_fact_sheet()` never printed it. Since both the Investment-Read GPT
prompt (`_prompt_payload`) and chat (`server._chat_system`) build their context
by calling `render_fact_sheet`, this one omission meant neither surface could
ever see or act on a signal that already existed. Added a `DATA CONFIDENCE` line
to `render_fact_sheet`, plus a hard rule in both `interpretation._SYSTEM_PROMPT`
and `server._chat_system`: hedge T2-unfooted figures instead of stating them with
T1 certainty.

# Part III — checking GPT's own arithmetic, without trusting it blindly  ·  8e7a0ad

`build_investment_read` already hands GPT binding guardrails and forbids it from
inventing or recomputing numbers — but nothing ever checked that the prose it
returned actually honored that. Added `_check_numeric_grounding`: after GPT
narrates, pull every `$`/`%` figure out of both its prose and the fact sheet,
normalize formatting (`$1.2M` ≈ `$1,200,000`, `17.5%` ≈ `17.50%`, `1,200` ≈
`1200`), and log a warning — not a block — for anything GPT stated that doesn't
trace back. Tested against a fabricated narration (`$999.0M` refi proceeds,
`55.0%` mezz piece neither of which were in the fact sheet): flagged exactly
those two, no false positives on the real, matching figures in the same text.
Warning-only by design — observability for how often the narration layer drifts,
not a gate on the response.

# Part IV — chat that's read its own memo  ·  8e7a0ad

`/api/chat` built its system prompt from the fact sheet alone; the Investment
Read narrative it generates at analyze-time was returned to the client and then
discarded — chat had no way to know what it had already told the user. Now
`_SESSIONS[sid]` keeps `read_md` alongside `fact_sheet`, and `_chat_system`
folds it into the prompt with an explicit instruction to stay consistent with it
and build on it rather than restate it.

# Part V — dead code sweep  ·  8e7a0ad
