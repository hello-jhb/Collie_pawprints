`static/index.html` went through several iterations in one sitting:

**Visual pass, matched to `collie.works`.** Pulled the real site's CSS (dark
page background `#050608` → `#0A0D10` → `#101318` gradient with a soft radial
highlight, white cards, JetBrains Mono for the wordmark/headings, Inter-style

**Structural rebuild.** The single biggest change: replaced the old two-page
flow (upload page → separate analysis page) with one persistent
`#appView` containing a skinny upload bar (two segments — underwriting model
/ optional operating statements — plus a compact Generate slot, all one row)
sitting permanently above the analysis, instead of the upload UI
disappearing once you generate. Iterated on card width from a first pass at
720px up to 1080px once real content (facts card line-wrapping, an empty-
looking chat panel) showed 720–840px was too tight for two side-by-side
columns.

**Gate added.** New `#accessView` — "Early Access" with Name / Work Email /
Company, two required checkboxes (prototype acknowledgment, upload
permission), and a `Continue` button gated on all five — sits in front of the
app now, per a request to move away from a bare upload-first landing page.

**Analysis Page UX Refresh (the last major pass).** The report was reading
like a generated summary; reworked it to read like an investment-committee
memo opener:
- `interpretation.py`'s GPT system prompt now emits **four** markdown
  sections instead of three: `## Overall Assessment` (new — a one-line bold
  verdict, meant to be the single most important line in the read) →
  `## Executive Summary` (renamed from "Investment Snapshot," instructed to
  narrate current state/what changed rather than restate deal basics the
  underwriting already established) → `## Key Findings` (renamed) →
  `## Areas for Review` (renamed from "Attention & Recommendations,"
  reframed explicitly as pointers to investigate, not directives).
- New `_overall_assessment()` in the deterministic (no-API-key) fallback
  derives the verdict from `fs["performance"]["noi_variance_pct"]` against
  the *same* `_NOI_VARIANCE_GATE` the `gap_driver` claim already uses, so the
  fallback path can't contradict the claims path.
- Front end: removed the permanent right-side chat rail — analysis is now
  full-width and single-column. Chat lives in a slide-out drawer
  (`#chatDrawer`, backdrop + translateX transition) triggered by a
  "💬 Ask questions about this investment" button at the foot of the report.
  The Facts card ("Investment Summary") moved out of the main flow and into
  the same `<details>` as Engine Detail, now labeled "Supporting facts &
  engine detail" — supporting evidence stays available but stops competing
  with the judgment for space.
- Fixed a real, unrelated bug surfaced while touching this: the progress
  spinner and the Generate button's "Reading…" label were reset back to
  normal on the *error* path in `analyze()` but never on success — a second
  deal would render with the button still saying "Reading…" and disabled.
  `renderDeal()` now resets both, and `#dealView` fades in via an opacity
  transition instead of an instant `display:none → block` flip (the
  "flicker" that was reported).

# Backend: variance narrative now says *when* and *whether it was budgeted*

The actual feature ask: the performance narrative could already say a
category ran over/under plan, but had no way to say *which month(s)* drove
it, and a small-dollar unbudgeted line (bad debt) was getting silently
dropped from the flagged findings because it didn't make the top-6 movers by
size.

- `perf_vs_plan_engine.py::build_variance_items` no longer collapses each
  expense leaf's monthly series into a single scalar before comparing plan
  vs. actual — it now also tracks `actual_by_period` / `plan_by_period` per
  concept. New `_concentrated_months()` picks the fewest chronological
  months whose combined magnitude clears a 75% share of a category's total,
  returning `None` when a cost is spread evenly across the year (so routine
  recurring costs never get a fabricated "concentrated in March" claim).
  Each row in `categories`/`movers`/`orphans` now carries `actual_months` /
  `plan_months`.
- `interpretation.py`: new `_period_phrase()` turns `["2021-05","2021-06"]`
  into `" in May and June 2021"`. The `operational_flags` claim now appends
  a period and, when a category has `status == "actual_only"`, a
  `" — not found in the underwriting's OPEX budget"` tag. Added a
  marketing/leasing branch (the original prompted example) and a generic
  fallback for any other unbudgeted category.
- **The real bug this surfaced:** `operational_flags` only ever looked at
  `items["movers"]` (top 6 by `abs(delta)`). Bad debt ($25K) ranked below
  six other categories with bigger dollar deltas and was silently excluded
  from the flags entirely, despite being flagged in `orphans` (categories
  with no match on one side). Fixed by unioning `movers` with `orphans`
  (deduped by concept) before iterating — an unbudgeted category is worth
  flagging regardless of how it ranks by size.

**Verified against real data**, not synthetic fixtures: ran
`perf_vs_plan_engine.build_perf_vs_plan()` and
`interpretation._performance_claims()` directly against
`Retail-Acquisition Underwriting.xlsx` / `Financial Statement 2021.xlsx`
(the same pair `test_perf_vs_plan_engine.py` already uses). Output now reads:

> Bad debt is +$25K over plan in March 2021 — not found in the underwriting's
> OPEX budget — read as a collections/credit trend, not a one-off; watch
> tenant health.

`test_perf_vs_plan_engine.py` (all cases, including the pre-existing "an
orphan is surfaced ... e.g. bad debt" check) still passes unchanged.

# Environment note: the preview dev server couldn't be launched this session

`mcp__Claude_Preview`'s `preview_start` failed all session long trying to run
`uvicorn server:app` — first `Path.cwd()` raising `PermissionError` with
`--reload`, then the same error class surfacing inside `importlib`'s
`_path_importer_cache` for every import attempted (including pure-Python
`h11`, and even a bare `os.getcwd()` call inside `http.server`'s own argparse
default). Confirmed this is specific to whatever sandbox the Preview tool's
subprocess spawner runs under this session — the identical
`uvicorn server:app --reload` command works fine when launched via the
regular Bash tool. Net effect: front-end changes this session were verified
by structural review (balanced tag counts, id cross-references, CSS rule
audits) and the user's own "Launch preview panel," not a live browser driven
by Claude. Backend changes (`perf_vs_plan_engine.py`, `interpretation.py`)
*were* verified directly via the Python REPL against real files, so that
half of today's work has actual execution proof; the HTML/CSS half does not.
